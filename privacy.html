Политика конфиденциальности приложения Threads-бот

Дата обновления: 2026-07-12

Это приложение — внутренний инструмент, который использует Светлана Фёдорова для публикации собственных постов и ответов в Threads (@svetlanafedorova_psy) через официальный Threads API компании Meta. Приложение не является публичным сервисом, не собирает данные сторонних пользователей и не передаёт доступ третьим лицам.

Какие данные обрабатываются

  • Access-токен Threads-аккаунта @svetlanafedorova (для публикации от имени владельца аккаунта через официальный OAuth-флоу Meta)
  • Тексты постов и комментариев, которые владелец аккаунта публикует через приложение
  • Публичные метрики собственных постов (просмотры, лайки, ответы, репосты) — для внутренней аналитики
  • Публичные тексты сторонних постов в Threads, найденные по ключевым словам, используются исключительно для подготовки черновиков ответов и не сохраняются дольше срока, необходимого для одобрения черновика владельцем аккаунта

Для чего используются данные

Исключительно для автоматизации публикации собственного контента владельца аккаунта и подготовки аналитических отчётов о его же публикациях. Приложение не собирает и не обрабатывает персональные данные посетителей сайта или подписчиков — только контент, который сам владелец аккаунта решает опубликовать.

Где и как хранятся данные

Данные хранятся на частном сервере (VPS), доступ к которому есть только у владельца аккаунта и технического администратора приложения. Токены доступа не передаются третьим лицам и не используются вне указанных выше задач.

Передача данных третьим сторонам

Для генерации черновиков текста используется API Anthropic (Claude) — в него передаются только тексты постов (собственных и найденных публичных) для обработки языковой моделью, без персональных идентификаторов. Ни один из сторонних сервисов не получает доступ к Threads-токену.

Хранение и удаление данных

Данные хранятся до тех пор, пока используется приложение, либо до запроса на удаление. Порядок удаления данных и отзыва доступа приложения описан на отдельной странице: инструкция по удалению данных.
